# Planning Rules

## When To Open
Use for discovery, estimates, architecture choices, and execution plans.

## Rules
1. Define goal, non-goals, and acceptance criteria before coding.
2. List constraints (time, dependencies, compatibility, risk).
3. Compare at least two implementation options for non-trivial work.
4. Choose the lowest-risk path that meets acceptance criteria.
5. Break work into independently verifiable steps.
6. Call out unknowns and how to resolve each one.

## Output Format
- Problem statement
- Proposed approach
- Step-by-step plan
- Risks and mitigations
- Verification plan
