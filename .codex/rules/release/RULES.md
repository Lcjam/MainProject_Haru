# Release Rules

## When To Open
Use when preparing merge, deployment, or rollback plans.

## Rules
1. Confirm change log / release notes reflect user-visible changes.
2. Validate migration and compatibility steps (if any).
3. Define rollback steps before release.
4. Confirm monitoring/alerts for affected services.
5. Time releases to reduce blast radius when possible.

## Release Readiness
- Build and test status known.
- Deployment steps documented.
- Rollback owner and procedure confirmed.
