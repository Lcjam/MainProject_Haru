# Review Rules

## When To Open
Use before handoff, PR creation, or when reviewing others' changes.

## Rules
1. Review for correctness first, style second.
2. Check for regressions in adjacent flows.
3. Verify error handling, edge cases, and null/empty inputs.
4. Confirm tests match behavior changes.
5. Mark unresolved risks and required follow-ups explicitly.

## Reviewer Checklist
- Scope matches request.
- Diff is understandable and maintainable.
- Tests are sufficient for changed behavior.
- Rollback impact is known.
